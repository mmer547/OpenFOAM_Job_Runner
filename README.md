# WSL OpenFOAM Job Runner

Windows上でWSL (Ubuntu) のOpenFOAMジョブを管理・実行するGUIツール。

## 必要条件

- Windows 10/11 (WSL対応)
- WSL 2 with Ubuntu (デフォルトディストリビューション)
- WSL上のUbuntuにインストールされた OpenFOAM
- Python 3.12+

## インストール

```powershell
pip install -r requirements.txt
```

依存パッケージ: PySide6

## 起動

```powershell
python main.py
```

## 使い方

### ジョブの追加

1. ツールバーの **「+ ジョブ追加」** をクリック
2. フォルダ選択ダイアログで OpenFOAM ケースフォルダを選択
3. 実行コマンドを追加:
   - プリセットボタンから選択: `blockMesh + simpleFoam` / `simpleFoam` / `blockMesh`
   - または「+ 追加」ボタンで手入力
4. **「OK」** でジョブをキューに追加

### ジョブ一覧

各ジョブの情報がテーブルに表示されます:

| 項目 | 説明 |
|------|------|
| ID | 自動生成される一意識別子 |
| 名前 | ケースフォルダ名（変更可能） |
| ステータス | 色付きで状態を表示 |
| 経過時間 | 実行開始からの経過時間 |
| 完了時間 | 完了した時刻 |
| コマンド | 実行コマンド (先頭のみ表示) |

### ステータス

| 状態 | 色 | 説明 |
|------|----|------|
| 待機中 | ● 青 | キューに入っており、実行スロット空き待ち |
| 実行中 | ● 緑 | WSL上でコマンド実行中 |
| 完了 | ● 灰 | 全コマンドが正常終了 (exit code 0) |
| 失敗 | ● 赤 | いずれかのコマンドがエラー終了 |
| キャンセル | ● 橙 | ユーザーにより中断（現状未実装） |

### ログの確認

- ジョブ一覧で行をクリックすると、下段のログエリアに実行ログが表示
- 実行中のジョブを選択すると、リアルタイムでログが追記される

### ジョブの削除

1. 削除したいジョブの行を選択
2. ツールバーの **「- 削除」** をクリック
3. 実行中のジョブは削除できません（完了後のみ削除可能）

### 同時実行数制限

WSLのリソース制限のため、同時に実行するジョブ数を制限できます。

1. メニュー **「設定 → 同時実行数...」** をクリック
2. スライダー / スピンボックスで 1〜8 の範囲で設定
3. **「OK」** で適用

デフォルト: **2**

---

## スケジューラ動作詳細

1. ジョブ追加時は `待機中` 状態でキューイングされる
2. スケジューラは0.5秒間隔でキューを監視
3. 実行中ジョブ数 < 最大同時実行数 の場合、待機ジョブを自動起動
4. ジョブ内のコマンドは**逐次実行**:
   - 例えば `blockMesh` → `simpleFoam` の順に実行
   - いずれかのコマンドが失敗 (exit code != 0) すると以降のコマンドは実行せず `失敗` 状態に
5. ジョブ完了後、次の待機ジョブがあれば自動起動

## WSL連携詳細

### パス変換

Windowsの絶対パス (例: `C:\Users\user\OpenFOAM\case-01`) は
WSLパス (`/mnt/c/Users/user/OpenFOAM/case-01`) に自動変換される。

### コマンド実行

以下の形式でWSL上でコマンドを実行:

```
wsl.exe -d Ubuntu --cd /mnt/c/path/to/case -- bash -c "<command>"
```

### WSLの前提

- WSLディストリビューション名: **Ubuntu** (デフォルト)
- ディストリビューション名を変更する場合は `src/wsl_manager.py` および `src/scheduler.py` 内の `distro="Ubuntu"` を修正

---

## 注意事項

- アプリケーション終了時にジョブの状態は保存されません（メモリ上のみで管理）
- 実行中のジョブはアプリ終了後もWSL上では動き続けますが、GUI側では追跡できなくなります
- フォルダ選択時に確認は行いません。既存のケースフォルダを直接操作するため、事前にバックアップを推奨

## 開発

### ファイル構成

| ファイル | 役割 |
|----------|------|
| `main.py` | エントリポイント |
| `src/models.py` | `Job` データクラス、`JobStatus` 列挙型 |
| `src/wsl_manager.py` | WSL実行ラッパー、パス変換 |
| `src/scheduler.py` | ジョブスケジューラ (QThread ベース) |
| `src/app.py` | `QApplication` 初期化 |
| `src/ui/main_window.py` | メインウィンドウ |
| `src/ui/job_list_widget.py` | ジョブ一覧テーブル |
| `src/ui/submit_dialog.py` | ジョブ追加ダイアログ |
| `src/ui/settings_dialog.py` | 同時実行数設定ダイアログ |

### ビルド (exe化)

```powershell
pip install pyinstaller
pyinstaller --onefile --windowed --name WslJobRunner main.py
```

`dist/WslJobRunner.exe` が生成される。
